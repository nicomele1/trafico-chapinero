# Traffic Modeling in Chapinero

This repository contains an exploratory implementation of traffic-assignment and signal-timing ideas for a road network in Chapinero. The code is intended for mathematical and computational study. It is not a calibrated model of traffic in Chapinero and does not provide operational signal-control recommendations.

## Project goal

The goal is to keep a small, reviewable Python project that makes the main formulations explicit:

- road graphs as directed graphs;
- BPR travel-time costs;
- Wardrop user equilibrium through the Beckmann formulation;
- system-optimum assignment through total system cost;
- Price of Anarchy as a comparison between user equilibrium and system optimum;
- Webster signal-timing formulas applied to a simplified phase model.

The numerical notebooks are examples of these formulations on a constructed instance. They are not traffic forecasts.

## Repository structure

- `src/graph_utils.py`: graph loading, graph enrichment, and road-capacity lookup.
- `src/costs.py`: BPR travel-time cost, Beckmann integral, total system cost, marginal cost, and Pigou toll formulas.
- `src/assignment.py`: route helpers, route-edge incidence construction, and edge-flow aggregation.
- `src/webster.py`: Webster cycle length, green allocation, and delay formulas.
- `notebooks/01_download_graph.ipynb`: download and enrichment of the local road graph.
- `notebooks/02_wardrop_equilibrium.ipynb`: finite-route Wardrop / Beckmann assignment.
- `notebooks/03_social_optimum.ipynb`: finite-route system-optimum assignment.
- `notebooks/04_price_of_anarchy.ipynb`: comparison of UE and SO total costs for the constructed instance.
- `notebooks/05_webster_optimization.ipynb`: schematic Webster signal-timing calculation.
- `tests/`: sanity tests for the extracted Python functions.
- `data/`: local raw and generated data. See `data/README.md`.
- `notes/`: mathematical outline and LaTeX notes.

## Mathematical model

We model the road network as a directed graph `G=(V,E)`. For each edge `e in E`, let `x_e >= 0` denote edge flow and let `t_e(x_e)` denote travel time.

The BPR function is used as a phenomenological congestion model:

```text
t_e(x_e) = t_e^0 * (1 + alpha * (x_e / c_e)^beta)
```

Here `t_e^0 > 0` is free-flow travel time, `c_e > 0` is capacity, and the default parameters in the code are `alpha = 0.15` and `beta = 4`.

For a finite set of candidate routes, route flows induce edge flows through

```text
x_e = sum_p delta[e,p] * f_p,
```

where `delta[e,p] = 1` if edge `e` belongs to route `p`, and `0` otherwise.

The user-equilibrium notebook uses the Beckmann objective

```text
sum_e integral_0^{x_e} t_e(u) du.
```

The system-optimum notebook uses total system cost

```text
TC(x) = sum_e x_e * t_e(x_e).
```

The Webster notebook applies cycle, green-time, and delay formulas to simplified phase groups. The phase groups are inferred from graph geometry and modeled flows, not from observed signal plans.

## Data workflow

Notebook `01_download_graph.ipynb` uses OSMnx to obtain a road graph for Chapinero from OpenStreetMap data available at run time. It then enriches the graph with speed, travel-time, and capacity attributes.

Generated GraphML files, pickle files, notebook caches, and processed outputs are local artifacts and are not intended to be committed. The file `data/raw/chapinero_road_network.png` is a small static image of the road graph.

The later notebooks depend on files generated by earlier notebooks. If those files are missing, rerun the notebooks in order.

## Installation

Create and activate a virtual environment from the repository root:

```bash
python3 -m venv .venv
source .venv/bin/activate
python -m pip install --upgrade pip
python -m pip install -r requirements.txt
```

The dependencies are listed in `requirements.txt`. They include OSMnx, NetworkX, CVXPY, NumPy, pandas, matplotlib, GeoPandas, Shapely, Jupyter, and IPython kernel support.

## How to run

Start Jupyter from the repository root:

```bash
jupyter notebook
```

Run the notebooks in this order:

1. `notebooks/01_download_graph.ipynb`
2. `notebooks/02_wardrop_equilibrium.ipynb`
3. `notebooks/03_social_optimum.ipynb`
4. `notebooks/04_price_of_anarchy.ipynb`
5. `notebooks/05_webster_optimization.ipynb`

Run the Python sanity tests with:

```bash
python -m unittest discover -s tests
```

## Limitations

This implementation is exploratory and is not calibrated against measured traffic counts.

Current limitations:

- O-D demands in the notebooks are synthetic unless explicitly documented otherwise.
- Route choice is restricted to a small candidate route set.
- Edge capacities are assigned by road-class heuristics.
- BPR parameters are defaults, not locally calibrated estimates.
- Solver status and numerical accuracy must be checked before interpreting UE, SO, or PoA values.
- Webster calculations use simplified phase grouping and do not include observed signal plans, turning counts, saturation-flow measurements, pedestrian phases, or signal coordination.
- Generated local files are required for the full notebook sequence but are not all committed.
